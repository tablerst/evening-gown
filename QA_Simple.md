## 晚礼服店铺网页设计需求QA提纲

### 一、项目与业务基本情况

1. 先简单介绍一下店铺和业务（成立时间、城市、线下门店数量等）
2. 目前主要提供哪些服务？
   - 仅租赁 / 仅定制 / 租赁 + 定制 / 现货销售等
3. 做官网/网页的**核心目的**是什么？
   - 提升品牌形象
   - 获取更多咨询/预约
   - 增加门店到店量
   - 推广新品或主打系列
   - 其他：**\_\_\_\_**

---

### 二、目标客群与使用场景

1. 理想中的主要客户是谁？
   - 新娘 / 伴娘
   - 晚宴、年会参加者
   - 走秀/舞台表演者
   - 私人高端定制客人
   - 其他：**\_\_\_\_**
2. 他们大致的年龄、消费水平和职业分布是怎样的？
3. 通常在什么**场景/节点**会想到来租/定制晚礼服？
   - 婚礼 / 订婚
   - 晚宴 / 年会 / 酒会
   - 公益活动 / 红毯 / 颁奖礼
   - 毕业舞会 / 生日派对等
4. 最希望通过网站吸引到哪一类客群？是否有**优先顺序**？

---

### 三、品牌定位与风格气质

1. 请用 3～5 个关键词形容品牌的气质（如：奢华、法式优雅、现代极简、复古、少女感…）。
2. 品牌目前在市场上希望被怎样认知？（如「中高端精品」「高定奢华」「新锐设计师品牌」等）
3. 品牌有没有已经确立的：
   - 品牌故事 / 品牌理念 / 口号
   - LOGO、标准字、标准色
   - 线下门店空间风格（比如：欧式宫廷/现代简约）
4. 希望网页给访客的第一印象是怎样的？
   - 「高级、稳重」还是「浪漫、梦幻」？
   - 更偏「时尚大片」还是「温暖真实的生活感」？

---

### 四、视觉与色彩偏好

1. 网站整体色彩倾向希望是？
   - 深色系（黑金、深蓝、墨绿…）
   - 浅色系（米白、香槟金、粉金…）
2. 有没有**必须使用**或**不能使用**的颜色？
3. 是否有现有物料（名片、价目表、店内装潢、海报）希望网页风格与之保持统一？
4. 有没有特别喜欢或特别不喜欢的**参考网站**？
   - 请提供 2～3 个链接，并简单说说喜欢/不喜欢的点（如：排版、色彩、动效、字体等）。

---

### 五、产品与礼服展示需求

1. 目前主要有哪些礼服类别？
   - 婚礼主纱 / 敬酒服 / 伴娘服 / 晚宴礼服 / 舞台礼服 / 礼服配饰等
2. 是否需要在网站中支持以下哪些**筛选维度**：
   - 颜色、尺码、风格（鱼尾、蓬裙、拖尾、礼服长度等）
   - 适用场景（婚礼、年会、红毯、舞台…）
   - 价位区间
3. 希望重点推哪些「明星款 / 爆款」礼服？是否有固定的**主打系列**？
4. 目前是否已经准备好：
   - 高清平面图（正面/侧面/背面）
   - 模特上身大片
   - 细节特写图（面料、刺绣、手工细节等）
5. 对展示形式是否有期望？
   - 基础图文展示即可
   - 需要 360° 旋转视图（可能对手机端不友好）
   - 需要走秀或动态试穿视频（可能对手机端不友好）

---

### 六、场景故事与搭配灵感

1. 是否希望网站中有专门的「场景搭配灵感 / 故事区」？
   - 例如：
     - 新娘 × 婚礼场景搭配建议
     - 年会女王造型参考
     - 舞台 / 红毯造型案例等
2. 是否有过往的真实场景照片或案例故事，方便整理为「搭配指南 / 穿搭灵感」？
3. 对内容形式的偏好：
   - 短文 + 大图
   - 图文教程型（步骤、tips）
   - 专栏文章 / 博客形式
4. 此模块的目标更偏向？
   - 做品牌形象与品味输出
   - 引导客户「主动来咨询」

---

### 七、顾客出片与口碑展示

1. 是否有「顾客穿着礼服的实拍照片」可以用于展示？
   - 是否需要做打码或隐私保护？
2. 是否希望网站中有：
   - 顾客出片展示区（大片墙）
   - 文本形式的顾客评价/好评截图
   - KOL / 博主 / 名人合作案例
3. 是否考虑开放某种程度的「用户投稿」？例如：
   - 顾客通过表单/社交媒体私信投稿照片
4. 对**社交媒体联动**有什么需求？
   - 显示小红书 / 抖音 / 微信视频号内容
   - 支持一键分享链接/图片到微信/微博等平台

---

### 八、服务流程与线上功能

1. 目前的**租赁流程**（从咨询到还衣）：
   - 一般包含哪些步骤？（咨询 → 选款 → 试穿 → 付款 → 使用 → 归还 → 押金处理……）
2. 如果有**定制服务**，定制流程与预计周期是怎样的？
3. 希望网站上哪些流程可以**线上完成或部分完成**？
   - 在线选款 + 提交意向单
   - 在线预约试穿时间/门店
   - 在线支付定金或全款
   - 在线签署租赁条款 / 注意事项
4. 是否已有使用中的系统需要对接？
   - 预约系统
   - CRM / 顾客管理
   - 支付系统（微信/支付宝/银行卡）
5. 是否需要**在线客服/咨询**？
   - 微信、企业微信、微博私信、电话还是第三方客服工具？

---

### 九、内容板块规划（页面结构层面）

1. 除了首页，希望一定要有哪些页面/栏目？（可多选）
   - 关于我们 / 品牌故事
   - 礼服系列 / 单品详情
   - 场景故事 / 搭配灵感
   - 顾客出片 & 口碑
   - 租赁流程 & 定制流程说明
   - 价格说明 / 常见问题（押金、破损赔付、改码等）
   - 联系我们 / 到店信息 / 门店环境
   - 招商合作 / 加盟信息
   - 招聘信息
2. 有没有必须放在首页首屏或非常靠前的位置的内容？（如：主打礼服系列、品牌 Slogan、预约入口等）
3. 是否需要多语言版本？（例如：中文 + 英文）

---

### 十、到店信息与门店展示

1. 目前有几家门店？每家门店的地址、营业时间和联系方式分别是什么？
2. 是否需要在网站上展示：
   - 门店实景照片 / 环境视频
   - 路线指引（地铁、公交、停车场提示等）
   - 地图导航（高德/百度/腾讯地图嵌入）
3. 是否有针对不同门店的**差异化服务或主打内容**？（如：某店专做婚礼主纱、某店以舞台礼服为主等）

---

### 十一、运营与营销需求

1. 是否有在做：
   - 小红书 / 抖音 / 视频号 / 微博 等运营？需要在哪些位置做引导？
2. 希望通过网站收集哪些客户信息？（姓名、电话、婚期/活动时间、预算区间等）
3. 未来是否会考虑：
   - 网站上的活动页 / 限时优惠页
   - 会员系统（积分、等级、回访提醒等）
4. 对搜索引擎优化（SEO）有什么期待？
   - 希望被哪些关键词搜索到？（例如「城市 + 晚礼服租赁」「婚纱礼服定制」等）

---

### 十二、素材准备

1. 目前已经有哪些现成的**素材**？
   - LOGO / VI 手册
   - 高清礼服图片、模特大片
   - 视频素材（走秀/幕后花絮/试穿过程等）
   - 文案/品牌故事/获奖经历等

---

### 十三、预算、周期与后续维护

1. 期望在什么时间上线？有没有必须配合的节点（如品牌发布会、店庆、节日档期等）？
2. 对本次项目的预算范围大致是？（可选填区间）
3. 上线后的**更新频率**大致如何？
   - 新款上新频率
   - 活动信息更新
4. 是否需要：
   - 后台管理系统，自行上传/下架礼服
5. 有没有对未来功能迭代的规划？（如后续开通线上商城、会员系统、积分等）
